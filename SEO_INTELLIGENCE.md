## 4. SCHEMA MARKUP OPPORTUNITIES

### Property Listing Pages (`/property/[slug]`)
```json
{
  "@type": "RealEstateAgent",  // closest available; or use Product
  "@type": "Product",
  "name": "9342 Longacre Dr, Detroit MI",
  "offers": {
    "@type": "Offer",
    "price": "50.12",
    "priceCurrency": "USD",
    "availability": "https://schema.org/InStock"
  },
  "aggregateRating": {
    "@type": "AggregateRating",
    "ratingValue": "82",
    "bestRating": "100",
    "ratingCount": "1"
  }
}
```
**Also apply:**
- `Place` schema with address, geo coordinates
- `FinancialProduct` (investment product) — use carefully; `Product` is safer
- `FAQPage` schema on any property FAQ section

### Comparison Pages (`/compare`, `/realt-vs-lofty`)
```json
{
  "@type": "Article",
  "articleType": "ComparisonArticle",
  "about": [
    {"@type": "WebSite", "name": "RealT", "url": "https://realt.co"},
    {"@type": "WebSite", "name": "Lofty", "url": "https://lofty.ai"}
  ],
  "mainEntity": {
    "@type": "FAQPage",
    "mainEntity": [/* FAQ pairs */]
  }
}
```
**Also apply:**
- `Table` markup via standard HTML table (Google reads tables for featured snippets)
- `Review` schema if Brickwise includes editorial scoring of platforms
