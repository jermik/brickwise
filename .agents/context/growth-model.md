# Brickwise Growth Model

*How we grow. Load before any distribution, SEO, or content agent runs.*

Growth mode declared 2026-05-09. Focus: content + distribution. SEO infrastructure work is paused unless explicitly unblocked.

4. **Indie Hackers + X**

5. **YouTube Shorts**
   - Single-property breakdowns, anchor on contrarian findings ("30.9% yield and I'd still avoid it")
6. **Email newsletter**
   - Primary conversion goal. Weekly cadence once subscriber base exists.

## Channels we are NOT using
- Paid ads (not until organic is validated)
- Affiliate co-marketing with Lofty/RealT (would compromise independence)
- Crypto influencer partnerships
- Mass email blasts to scraped lists

## Conversion ladder
1. SERP click or LLM citation
2. Lands on score page or city page
3. Reads methodology
4. Subscribes to newsletter (primary KPI)
5. Returns for next week's data drop
6. Clicks affiliate link on platform-decision moment

## Content shape that works for Brickwise
- Specific property breakdowns by address with score + risk
- City-level rollups with concentration warnings
- Contrarian findings (high-yield trap teardowns)
- Cross-platform comparisons (Lofty vs RealT for specific job-to-be-done)
- Methodology explainers (one weight per post)

## Content shape that does NOT work for Brickwise
- Generic "what is tokenized real estate" 101 posts (commodity content)
- "Top 5 tokenized properties" without methodology
- AI-trend tie-ins ("How AI is changing real estate")
- Personal-brand founder posts disconnected from data

## Distribution rules
- Every piece adapted per channel before posting
- No link drops without context
- One specific number in the first 100 characters
- Every external post links to the same canonical page on brickwise (not a duplicate)
