# FactuurDirect UX Philosophy

## Core principles
1. **Contextual hints, not hidden automation.** The product never decides quietly. It shows the decision and lets the user override.
2. **Predictable behavior.** Same input, same output, every time.
3. **Explicit user control.** No background "smart" actions that change records.
4. **Low cognitive load.** Default values picked carefully, fields surfaced only when needed.
5. **Time-to-first-invoice is the ranking metric.** Anything that delays it is suspect.

## Visual rules
- Calm aesthetic: spacious layouts, neutral palette, no neon
- One primary CTA per screen
- Status communicated in plain words ("Verzonden", "Concept"), not icon-only
- Mobile-first, especially the invoice creation flow

## Interaction rules
- BTW selection visible, with one-line explanation of why it was picked
- Edits are undoable
- Saving and sending are distinct, intentional actions
- No surprise navigations after an action

## Tax + workflow rules
- KOR, ICP, reverse charge surfaced with plain-Dutch explanations
- Customer country drives BTW logic and the product explains the decision inline
- Invoice numbering predictable, user-controllable

## What we do not do
- Gamify invoicing
- Add a feed or social layer
- Add a chatbot
- Add "AI insights" widgets
- Hide cost previews behind a click
