## Spot-check: Image Dimensions / Layout Shift / Font Display

- **Font display**: All three Google Fonts (Inter, DM_Mono, DM_Serif_Display) loaded with `display: "swap"` ✅. Third font (General Sans from fontshare) loaded non-blocking via `media=print` swap trick with `display=swap` in URL query param ✅.
- **Image dimensions**: `FallbackImg` renders `<img>` without explicit `width`/`height`. Callers on homepage and property pages do not pass these props. Container CSS provides height but browsers cannot reserve layout space without native dimensions. Flagged as Finding #4.
- **Layout shift hints**: No autoplay, no render-blocking CSS beyond the non-blocking pattern in layout. CLS risk is isolated to `FallbackImg`.
