**Additional observation (homepage + property pages)**: `FallbackImg` component (`components/ui/fallback-img.tsx`) renders a plain `<img>` without explicit `width`/`height` HTML attributes. Dimensions are CSS-only (`w-full h-full`, `w-10 h-10`). This can contribute to CLS on the LCP image. The container divs do have explicit pixel dimensions (`sm:w-[260px] h-[200px]`), which mitigates but does not eliminate the risk. **Not whitelist-fixable** — lives in `components/ui/`.

**Font loading**: Inter, DM Mono, DM Serif Display loaded via `next/font/google` with `display: 'swap'` — correct. General Sans loaded via FontShare using a non-blocking media=print swap pattern — correct.

Sitemap routes found in `app/sitemap.ts`:
- Static: 19 routes (homepage, compare pages, learn pages, platform pages, about, contact, privacy, algorand, market)
- Dynamic: city routes, ranking routes, property routes (~460), market update routes, algorand project routes (33)
- **MISSING**: `/methodology` — see finding #22

The sitemap includes `new Date()` for `lastModified` on all static routes. This will make every static URL appear freshly modified on every sitemap fetch — minor concern (Googlebot may deprioritize crawls when lastmod is always "today"). This is an architectural decision, not a content fix, so flagging as informational only.
