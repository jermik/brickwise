## Out of Scope — Needs Human

1. **`/analyzer` page has no `metadata` export** — the file uses `"use client"` directive which prevents server-side metadata export in Next.js App Router. Fixing this requires splitting the page into a server-component wrapper (exports metadata) + a client-component inner file. This is an architectural change outside the whitelist.

2. **Live URL verification blocked** — brickwise.pro returns 403 from the sandbox environment (likely Vercel preview protection or IP-based access control). Cannot verify rendered `<title>`, canonical, h1, OG tags, JSON-LD, or font loading behaviour from live HTML. A human should run the monitor checks against the live URL or configure sandbox network access.

3. **OG images on compare pages** — while adding `images` to the OG metadata block is whitelist-safe (it's inside `metadata` on a PublicShell page), the actual `/opengraph-image` route should be verified to work correctly before adding the tag. Human should confirm the image route returns valid OG image content for compare pages.

4. **Image dimension audits and CLS** — cannot check without live page access.

5. **`/privacy` lacks JSON-LD** — low priority but worth flagging for completeness.
