## Spot-checks: Fonts, Images, Layout Shift

**Fonts:** Inter, DM_Mono, and DM_Serif_Display all use `display: "swap"` in `layout.tsx`. GeneralSans from fontshare.com loaded via `NonBlockingStylesheet` (media=print trick, non-render-blocking). `display=swap` passed in the CSS URL. ✓

**Images (potential CLS):** `FallbackImg` component (`components/ui/fallback-img.tsx`) is a plain `<img>` wrapper with no explicit `width` or `height` attributes set by default. On the homepage "Best pick today" card, the image container uses `sm:h-auto`, meaning the height is not constrained on sm+ breakpoints. Without dimensions on the `<img>`, there is a potential CLS risk on tablet/desktop when the image loads. Not critical (container flex layout helps), but worth noting.

**Image dimensions:** The `FallbackImg` usages on property cards pass `className="w-full h-full object-cover"` inside fixed-height containers, which constrains the layout. CLS risk is LOW but present.
