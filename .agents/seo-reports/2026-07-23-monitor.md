## Notes

### Methodology page sitemap gap (Finding #1)
`app/methodology/page.tsx` — confirmed PublicShell, confirmed exports metadata, confirmed canonical `https://brickwise.pro/methodology`. Not present in any `staticRoutes` entry in `app/sitemap.ts`. This page explains scoring weights (30/25/20/25), which is a core trust signal and conversion lever. It should be discoverable.

### Analyzer page — false alarm
`app/analyzer/page.tsx` is `"use client"` and cannot export metadata directly. This is correctly handled via `app/analyzer/layout.tsx`, which exports full metadata including twitter card, OG image, canonical, and JSON-LD. Not an issue.

### Title truncation — context
Google's character limit is ~50–60 display chars (pixel-width based). The monitor uses raw char count as a proxy. Em-dash characters (`—`) count as ~1 character but render wider. Titles shown may be truncated sooner than the char count alone suggests.

### Robots.txt — baseline established
No prior report to diff against (first run). Current state:
- Disallows: `/dashboard`, `/portfolio`, `/watchlist`, `/settings`, `/sign-in`, `/sign-up`, `/announce`, `/seo-dashboard`, `/crm`, `/growthos`, `/api/`
- Sitemap: `https://brickwise.pro/sitemap.xml`
- Host: `https://brickwise.pro`
- Status: appropriate and correctly blocks all auth-gated + CRM routes.

### Live 403 block
All WebFetch requests to `brickwise.pro` returned HTTP 403 Forbidden. The site's CDN (likely Cloudflare) blocks the agent sandbox proxy IP range. Findings above are from static source analysis only. Tags that are generated dynamically (e.g., JSON-LD with live property counts) could not be rendered-output verified.

### Spot-check: images, fonts, layout shift
- **Image dimensions:** No `<img>` or `<Image>` tags found without explicit `width`/`height` in the three spot-checked pages (`/`, `/learn/lofty-review`, `/learn/realt-review`). OK.
- **Font-display:swap:** All next/font/google fonts (Inter, DM Mono, DM Serif) correctly set `display: "swap"`. Fontshare CSS URL also includes `display=swap` parameter. OK.
- **Layout shift indicators:** No auto-playing media or blocking script patterns observed in spot-checked pages.

### JSON-LD
Homepage (2 blocks), `/compare/realt-vs-lofty` (3 blocks), `/learn/lofty-review` (4 blocks), `/learn/realt-review` (3 blocks), `/learn/what-is-tokenized-real-estate` (3 blocks), `/property/[id]` (1 block), `/city/[city]` (3 blocks), `/rankings/[category]` (2 blocks), `/analyzer` (2 blocks via layout). All pass static JSON-LD parse check (valid object literals in source).

### H1 tags
All 26 checked PublicShell pages have exactly one `<h1>` tag. OK.

### Canonical links
All 26 PublicShell pages export `alternates.canonical` pointing to the correct absolute URL. OK.

### Open Graph
All pages have `openGraph` metadata. Six pages lack an explicit `og:image`, relying on the root-level `app/opengraph-image.tsx` fallback (which Next.js serves at `/opengraph-image`). This is acceptable behavior — the fallback image is served.
