## Out of scope — needs human

### 1. `app/analyzer/page.tsx` — Missing metadata (structural)

The page is `"use client"` at the top level, meaning it cannot export Next.js `metadata`. No metadata currently served for `/analyzer`. Fix requires:
- Extract interactive logic into a new `AnalyzerClient.tsx` client component
- Convert `app/analyzer/page.tsx` to a server component that exports `metadata` and renders `<AnalyzerClient />`

This is outside the whitelist (creating a new component file).

**Suggested metadata** (for human to apply):
```ts
export const metadata: Metadata = {
  title: { absolute: "Property Analyzer — Filter 460+ Tokenized Rentals | Brickwise" },
  description: "Filter and sort every tokenized rental property on Lofty and RealT by yield, risk, score, and city. Free, no signup, updated daily.",
  alternates: { canonical: "https://brickwise.pro/analyzer" },
  openGraph: { title: "Tokenized Property Analyzer — Brickwise", type: "website", url: "https://brickwise.pro/analyzer" },
};
```

### 2. Potential double-branding in title template

Several pages use `title: "... | Brickwise"` as a plain string. The root layout defines `title: { template: "%s | Brickwise" }`. If Next.js applies the template to plain-string child titles, the rendered output would be `"... | Brickwise | Brickwise"`. Pages that correctly bypass this use `title: { absolute: "..." }`.

Verify in browser DevTools: if `/learn/lofty-review` shows "Lofty Review 2026: Yields, Fees & Risks | Brickwise | Brickwise" in the tab title, all plain-string titles need conversion to `{ absolute: "..." }`. This affects `app/page.tsx` and all learn/compare pages using plain string titles.
