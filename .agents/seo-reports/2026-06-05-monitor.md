## Spot-check: 3 pages for image dimensions, CLS, font-display

**Page 1 — Homepage (`/`)**
- `FallbackImg` (best-pick card): raw `<img>` with no `width`/`height` attributes. Container has `sm:h-auto` (auto height on ≥640px), meaning image height is indeterminate before load → potential CLS.
- `FallbackImg` (avoid-property cards): `h-[120px]` fixed container ✓ — no CLS risk on those cards.
- Fonts: Inter/DM_Mono/DM_Serif all use `display: "swap"` ✓. Fontshare via `NonBlockingStylesheet` (media=print swap) with `&display=swap` in URL ✓.

**Page 2 — Property detail (`/property/[id]`)**
- `PropertyHeroImage`: no `width`/`height` attributes. Container has fixed CSS height (`h-[280px]`/`sm:h-[340px]`) which bounds CLS — risk is low but not zero before CSS parse.
- Alt text: `alt={p.name}` uses property street address ✓.
- `<img>` for property in best-pick section: same container pattern.
- Fonts: inherit layout ✓.

**Page 3 — Algorand index (`/algorand`)**
- `ProjectLogo` component not examined in detail (in `components/algorand/`).
- Font: inherit layout ✓.

**CLS verdict:** `FallbackImg` and `PropertyHeroImage` in `components/ui/` and `components/property/` are blacklisted for this routine. The best-pick card on the homepage is the highest-impact CLS risk. Needs human to add `width`/`height` or an `aspect-ratio` wrapper.
