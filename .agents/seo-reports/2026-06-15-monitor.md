## Out of Scope ‚Äî Needs Human

### NH-01 ‚Äî Title double-suffix risk (cannot verify without live access)

Multiple pages use plain string titles that already include `| Brickwise`, while the root layout sets `template: "%s | Brickwise"`. In Next.js 13‚Äì15, the template is applied to plain string titles from child pages, which would produce `"...| Brickwise | Brickwise"` for all affected pages.

Affected plain-string titles containing `| Brickwise`:
- `app/page.tsx` ‚Äî "Tokenized Real Estate Scored ‚Äî Lofty & RealT | Brickwise"
- `app/learn/what-is-tokenized-real-estate/page.tsx`
- `app/learn/lofty-review/page.tsx`
- `app/learn/realt-review/page.tsx`
- `app/algorand/page.tsx`
- `app/rankings/[category]/page.tsx`

Since AGENTS.md flags Next.js 16 as having breaking changes, this behavior may differ. **Human should open these pages in a browser or Vercel preview and inspect `<title>` in devtools** to confirm. If double-suffix is confirmed, fix is to convert all affected titles to `{ absolute: "..." }`.

### NH-02 ‚Äî `/methodology` missing from sitemap

`app/sitemap.ts` does not include `/methodology` as a static route. If Googlebot follows internal links to this page, it will crawl it but it won't appear in the sitemap with a `lastmod` date. Fix: add `{ url: \`${base}/methodology\`, lastModified: new Date(), changeFrequency: "monthly", priority: 0.7 }` to `staticRoutes` in `app/sitemap.ts`. This is within the whitelist but was not discovered via live sitemap fetch, so a human should confirm the page is intentionally included (it has a PublicShell wrapper and Article + FAQPage schema, so it should be indexed).
