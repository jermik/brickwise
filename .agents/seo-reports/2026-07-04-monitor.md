## Summary

The codebase is structurally sound with consistent PublicShell usage across all indexable routes, proper canonical tags, and working JSON-LD blocks. Two systemic issues need attention:

1. **Double "| Brickwise" in rendered titles** — the root layout defines a `template: "%s | Brickwise"` that appends `| Brickwise` to any page whose `metadata.title` is a plain string. ~15 pages include `| Brickwise` at the end of their plain string title without using `{ absolute: "..." }`, which means Google sees titles like `"...| Brickwise | Brickwise"`. This is the highest-priority fix.
2. **Missing OG images on ~10 pages** — when a page defines its own `openGraph` block without an `images` key, Next.js App Router does NOT inherit the global `/opengraph-image` set in layout.tsx. These pages share with no thumbnail on social.
