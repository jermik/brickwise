## What's good

- All 26 PublicShell pages export metadata (except `/analyzer` which is architecturally blocked).
- All checked pages have exactly 1 `<h1>`.
- Root layout has `lang="en"` correctly set.
- All Google Fonts loaded with `font-display: swap` — no CLS font risk.
- External font (General Sans via Fontshare) loaded non-blocking via `media="print"` swap pattern.
- Auth-gated pages (`/portfolio`, `/watchlist`, `/sign-in`, `/sign-up`, `/crm`, `/seo-dashboard`) all have `robots: { index: false, follow: false }` and are also disallowed in `robots.ts`.
- JSON-LD present on most high-value pages: homepage (2 blocks), compare/realt-vs-lofty (3), learn/* (3–4), algorand (2), rankings (2), city (3), property/[id] (2).
- No images found without `alt` attributes in audited pages.
- All images use Next.js `<Image>` with `width`/`height` or `fill` — no CLS image risk.
- Internal linking structure present in learn and compare pages linking to /analyzer.

## Out of scope — needs human

1. **Analyzer SSR fix (#1 above)** — Requires structural change to how `/analyzer` is composed. The current `"use client"` root component cannot export server metadata. Options: (a) add `app/analyzer/layout.tsx` with static metadata, (b) wrap with a server component that renders a client island, or (c) accept that /analyzer is not SEO-optimized (it's a tool page; may be intentional).
2. **Live site 403 (#9 above)** — Cannot confirm what Googlebot actually sees without resolving bot protection or testing from a clean IP. Recommend manual verification of rendered HTML for canonical, JSON-LD, and title tags.
