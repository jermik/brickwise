## Pattern confirmed and fixed

All 17 HIGH-severity title violations and the MEDIUM description issue were in the whitelist and fixed. Each fix uses `title: { absolute: "..." }` which:
1. Locks the title string at ≤60 chars
2. Prevents the root-layout `%s | Brickwise` template from double-appending "| Brickwise" if Next.js 16 applies it to nested routes

## Out of scope — Needs human

1. **`/market/[date]` titles** — dynamically generated from `market-updates.json`. Need human to audit each `title` field in that JSON for length > 48 chars, and update `generateMetadata` to use `{ absolute }`.

2. **Next.js 16 template behavior** — Verify whether plain `title: "string"` in nested route segments inherits the root `template: "%s | Brickwise"`. If yes, pages like `city/[city]`, `algorand/[slug]`, `market/[date]` are rendering double "| Brickwise" in title tags. Run: `curl -s https://brickwise.pro/city/detroit | grep '<title>'` to check.

3. **`ProjectLogo` image layout shift** — `components/algorand/project-logo.tsx` renders images without explicit `width`/`height`, causing potential CLS. File is inside `components/` (blacklisted from routine edits).

4. **Analyzer `/analyzer` client-only H1** — `app/analyzer/page.tsx` is a `"use client"` component; H1 "Every tokenized rental, scored" only renders after JS hydration. Googlebot will eventually execute JS but the initial HTML has no H1 — consider adding SSR wrapper or static H1 in the layout for search engines.
